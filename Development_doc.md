# Frontend Guidelines (Quick Reference)

## Purpose
Simple rules for developers to follow while writing frontend code.  
Read fast, apply immediately.


### Service Guidelines

- Use proper naming for service.
- Use proper naming for function.
- Use proper naming for variable.


---

### Proto Guidelines

- Use `optional` for nullable fields
- Use single formrequest if possible(fileds are same).
- Use string for date and date time.
- Naming should be same as entity.
- Copy every field from backend to frontend.
